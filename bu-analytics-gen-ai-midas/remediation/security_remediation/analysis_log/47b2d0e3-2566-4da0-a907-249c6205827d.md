## Root cause
deploy/ecs-app/modules/ecr/main.tf line 19: the aws_ecr_repository resource uses image_tag_mutability = var.image_tag_mutability whose default allows MUTABLE tags. MUTABLE ECR tags mean a Docker image tag (e.g. 'latest') can be silently overwritten, enabling supply-chain attacks where a compromised tag silently replaces a production image. The encryption_type is AES256 (not KMS), so images are not protected by a customer-managed key.

## Remediation plan
1) deploy/ecs-app/modules/ecr/main.tf: change the default for image_tag_mutability to IMMUTABLE in variables.tf, or hard-code image_tag_mutability = 'IMMUTABLE' in the resource block.
2) Change encryption_configuration encryption_type to KMS and add kms_key = var.ecr_kms_key_arn, wiring it to the existing KMS key used by EKS.
3) Add a Checkov rule CKV_AWS_51 (ECR tag immutability) and CKV_AWS_136 (ECR KMS encryption) to CI.
