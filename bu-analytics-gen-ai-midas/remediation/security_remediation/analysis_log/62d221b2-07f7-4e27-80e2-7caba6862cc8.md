## Root cause
Dockerfile Misconfiguration: Dependency Confusion in backend/Dockerfile.graphrag line 10. Packages are installed from public registries (PyPI/npm) without hash pinning. A dependency confusion attack occurs when an attacker registers a public package matching a private package name with a higher version, causing pip/npm to silently install the malicious package.

## Remediation plan
1) For pip-based Dockerfiles: generate requirements with pinned hashes using pip-compile --generate-hashes; use pip install --no-deps -r requirements.txt.
2) For npm: use npm ci (not npm install) to enforce package-lock.json integrity.
3) Commit package-lock.json and keep it up to date.
4) Use ECR-mirrored package proxies where available inside the VPC.
5) Add Trivy or Snyk supply-chain scanning to CI.
