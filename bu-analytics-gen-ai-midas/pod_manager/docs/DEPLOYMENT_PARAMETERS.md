## ALB / Ingress listeners

Application pods (Envoy, router.svc, backend, login) **do not** toggle for port 80 vs 443. Only the Ingress layer controls external listeners.

| Parameter | Local | AWS | Source | Created by |
|-----------|-------|-----|--------|------------|
| `ingress.listeners.http.enabled` | N/A (host ports) | `true` (default) | Helm values | Operator |
| `ingress.listeners.http.port` | N/A | `80` | Helm values | Operator |
| `ingress.listeners.https.enabled` | N/A | `false` (default) | Helm values | Operator |
| `ingress.listeners.https.port` | N/A | `443` | Helm values | Operator |
| `ingress.listeners.https.certificateArn` | N/A | `""` until ACM/cert-manager | Helm / ACM | AWS ACM or cert-manager (future) |
| `ingress.listeners.https.sslRedirect` | N/A | `false` | Helm values | Operator |
| `alb.ingress.kubernetes.io/listen-ports` | N/A | Computed by Helm helper | [`_helpers.tpl`](../infra/helm/routing-tier/templates/_helpers.tpl) | Helm render |

Enable HTTPS later:

```yaml
ingress:
  listeners:
    http:
      enabled: true
    https:
      enabled: true
      certificateArn: arn:aws:acm:us-east-1:ACCOUNT:certificate/ID
```

## Postgres

Routing state lives in the shared backend Postgres (a local `postgres:16` container locally; RDS in real deploys), inside a dedicated `pod_manager` schema that isolates it from the backend's `public` schema. The schema and its `pm_`-prefixed tables are created automatically at startup via `CREATE SCHEMA IF NOT EXISTS` + `CREATE TABLE IF NOT EXISTS`, and the connection pool pins its `search_path` to that schema. Config lives in the `[postgres]` section of `app_config.toml` (`dsn`, `schema_name`, `table_prefix`, `pool_min`, `pool_max`, `command_timeout_sec`), overridable with `POD_MANAGER_POSTGRES_*` env vars.

| Parameter | Local | AWS | Source | Created by |
|-----------|-------|-----|--------|------------|
| `DATABASE_URL` | `postgresql://postgres:postgres@postgres:5432/midas` | RDS DSN (e.g. `postgresql://USER:PASS@<rds-endpoint>:5432/midas`) | Compose / Secret | `docker-compose.local.yml` / Secrets Manager |
| `POD_MANAGER_POSTGRES_SCHEMA_NAME` | `pod_manager` | `pod_manager` | Compose / Helm | Operator |
| `POD_MANAGER_POSTGRES_TABLE_PREFIX` | `pm_` | `pm_` | Compose / Helm | Operator |
| `POD_MANAGER_POSTGRES_POOL_MIN` | `app_config` default | same | `app_config.toml` + env | Operator |
| `POD_MANAGER_POSTGRES_POOL_MAX` | `app_config` default | same | `app_config.toml` + env | Operator |
| `POD_MANAGER_POSTGRES_COMMAND_TIMEOUT_SEC` | `app_config` default | same | `app_config.toml` + env | Operator |
| Physical table names | `pod_manager.pm_*` (`pm_backend_pool`, `pm_login_pod_pool`, `pm_user_assignments`, `pm_assignment_events`, `pm_solution_documents`, `pm_service_config`) | same | `[postgres]` in `app_config.toml` + env | router.svc at startup (`CREATE SCHEMA` + `CREATE TABLE IF NOT EXISTS`) |
| IRSA role | N/A | `pod_manager_irsa_role_arn` | Terraform output â†’ Helm `serviceAccount.roleArn` | Terraform `iam-pod-manager` |

## Terraform (AWS infrastructure)

| Parameter | Local | AWS | Source | Created by |
|-----------|-------|-----|--------|------------|
| `create_vpc` | N/A | `true` or `false` | `terraform.tfvars` | Operator |
| `existing_vpc_id` | N/A | VPC ID when `create_vpc=false` | `terraform.tfvars` | Existing AWS VPC |
| `existing_private_subnet_ids` | N/A | Private subnet IDs for EKS | `terraform.tfvars` | Existing AWS VPC |
| `eks_cluster_version` | N/A | e.g. `1.29` | `terraform.tfvars` | Terraform |
| `eks_node_instance_types` | N/A | e.g. `["t3.medium"]` | `terraform.tfvars` | Terraform |

When the AWS account is at **VPC quota**, set `create_vpc = false` and supply an existing VPC and private subnets (see [`terraform.tfvars`](../infra/terraform/environments/dev/terraform.tfvars)).
