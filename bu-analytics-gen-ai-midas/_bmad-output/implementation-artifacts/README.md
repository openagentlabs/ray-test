# Implementation artifacts

Stories and sprint tracking from `bmad-create-story` land here.

## Story template

Copy `templates/story-template.md` when authoring manually; prefer `bmad-create-story` in a fresh chat.

## Test gate

Stories for scenarios **1–3** must include **Test acceptance criteria**. Implementation is not done until `bmad-dev-story` (or `bmad-quick-dev`) runs those tests successfully. See `_bmad-output/testing/scenario-test-gate.md`.
