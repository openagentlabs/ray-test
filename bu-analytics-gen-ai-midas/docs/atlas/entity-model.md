# Entity model

Atlas has exactly four persisted entities. They share a common spine
(audit + soft-delete) defined by [`BaseEntity`](../../atlas/src/lib/db/entities/base/README.md).

## Diagram

```
                             ┌────────────────┐
                             │     User       │
                             └────────┬───────┘
                                      │ ownerUserId
                                      ▼
                             ┌────────────────┐
                ┌────────────│    Account     │────────────┐
                │            └────────────────┘            │
                │ accountId               accountId        │
                ▼                                          ▼
       ┌────────────────┐                        ┌──────────────────┐
       │     Login      │                        │     Config       │
       │ (discriminated │                        │ body: ConfigBody │
       │   by `kind`)   │                        └──────────────────┘
       └────────────────┘
```

`Account` is the central entity. `Login` and `Config` reference it via
`accountId`. `User` owns `Account` via `ownerUserId` and is also the
target of every `createdByUserId` / `updatedByUserId` pointer on every
record (including itself).

## Common spine — `BaseEntity`

Every persisted record carries:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | uuid v4 | yes | Generated by `crypto.randomUUID()` in the repository on `create`. |
| `createdAt` | iso datetime | yes | Set on `create`, never re-stamped. |
| `updatedAt` | iso datetime | yes | Re-stamped on every `update` / `softDelete` / `restore` / `setEnabled`. |
| `createdByUserId` | uuid | yes | The user who first inserted the row. `SYSTEM_USER_ID` for bootstrap rows. |
| `updatedByUserId` | uuid | yes | Re-stamped on every mutation. |
| `enabled` | boolean | yes | Defaults to `true`. Disabled rows are filtered out of `findAll` / `findById` by default. |
| `deleted` | boolean | yes | Defaults to `false`. Soft-deleted rows stay on disk for audit. |

## `User`

| Field | Type | Notes |
|---|---|---|
| `email` | email (lowercased on parse) | Unique across the user collection. |
| `displayName` | string (1–120) | Free-form human label. |

A user is a **principal**. How they prove identity is in `Login`, not
here. Reserved IDs:

- `SYSTEM_USER_ID = 00000000-0000-0000-0000-000000000000` — bootstrap
  audit-only user.
- `DEFAULT_ADMIN_USER_ID = 00000001-...-000000000002` — owner of the
  default account.

## `Account` (central)

| Field | Type | Notes |
|---|---|---|
| `ownerUserId` | uuid | Required. The user that owns this account. |
| `name` | string (1–160) | Mutable label (rename via `AccountRepository.update`). |
| `description` | string (≤2000), optional | Free-form notes. |
| `address` | `Address`, optional | Strict object: `line1`, `line2`, `city`, `region`, `postalCode`, `country` (each optional). |

Reserved ID: `DEFAULT_ACCOUNT_ID = 00000001-...-000000000001`.

## `Login` (discriminated union)

| Common field | Type | Notes |
|---|---|---|
| `kind` | discriminator | Today only `'login-email-password'`. New flavours plug in here. |
| `accountId` | uuid | The account this login authenticates against. |

### `'login-email-password'` variant

| Field | Type | Notes |
|---|---|---|
| `email` | email (lowercased on parse) | Unique per `(accountId, email)` (enforced by `LoginRepository.findEmailPassword`). |
| `passwordHash` | bcrypt string (cost 12) | NEVER store plaintext; use `LoginEmailPassword.hashPassword(plaintext)`. |
| `lastLoginAt` | iso datetime, optional | Updated by `LoginRepository.markLogin` on a successful verify. |

## `Config`

| Field | Type | Notes |
|---|---|---|
| `accountId` | uuid | The owning account. |
| `key` | string (1–120) | Profile name. Today always `'default'`; reserved for future named profiles. |
| `body` | `ConfigBody` (strict object) | The actual settings (`theme`, `git`, `integrations`, `ui`). Mirrors the legacy `AppConfig` minus the now-irrelevant `storageConsent`. |

`Config` replaces what used to live in browser `localStorage` under the
key `atlas.app-config.v1`. The `toAppConfig(body)` adapter sets
`storageConsent: 'granted'` when bridging back to the React tree, so
existing components keep working unchanged.

## Invariants enforced in code

| Invariant | Enforced by |
|---|---|
| `id` is uuid v4 | `BaseEntitySchema` (Zod). |
| `createdAt` ≤ `updatedAt` | `BaseRepository.create` sets them equal; `update` only ever moves `updatedAt` forward. |
| `createdByUserId` and `createdAt` are immutable | `BaseRepository.update` re-applies the original values. |
| `email` is lowercase | `UserSchema` and `LoginEmailPasswordSchema` use `.toLowerCase()` in Zod. |
| Plaintext passwords never persist | `LoginRepository.createEmailPassword` is the only public path to inserting an email-password login. |
| `Config.body` rejects unknown keys | `ConfigBodySchema.strict()`. |

## Acceptance criteria

A change to the entity model is "done" when:

1. The Zod schema and the data class agree (round-trip through
   `fromRecord` / `toRecord` is the identity).
2. The entity folder ships a `README.md` matching the layout of the
   existing entity READMEs.
3. The entity folder ships `__tests__/<Entity>.test.ts` covering: a
   complete-record parse, at least one rejection, and at least one
   class round-trip.
4. The entity is registered in `DatabaseSchema` (`atlas/src/lib/db/context/types.ts`)
   and the in-memory document compiles.
5. A concrete repository extends `BaseRepository` and is exposed via a
   `getXRepository()` singleton accessor.
6. A `__tests__/<Entity>Repository.test.ts` covers each entity-specific
   finder.
7. At least one server action is added under `actions/<area>/`,
   following the conventions in [`server-actions.md`](server-actions.md).
8. `npm test`, `npm run lint`, and `npm run build` are all green.
